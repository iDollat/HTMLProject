    • XML oznacza eXtensible Markup Language, czyli rozszerzalny język znaczników. Jest to specyfikacja dotycząca przechowywania informacji oraz opisująca strukturę tych informacji.


    XML (cechy / własności)…

    • przechowywanie danych,
    • prezentowanie danych,
    • przenoszenie informacji,
    • specyfikacja XML pozwala tworzyć własny język znaczników.

    Rozszerzenie pliku

    Plik zapisujemy z rozszerzeniem .xml. Pliki możemy otworzyć za pomocą dowolnego edytora tekstowego.


    Zastosowanie

    XML można łatwo przesyłać między komputerami i używać w dowolnym języku programowania.


    Walidacja

    Na stronie xmllint.com możesz zweryfikować poprawność pliku XML (to bardzo przydatne narzędzie).


    Prolog

    Pierwszy wiersz w pliku xml to prolog (nie posiada znacznika zamykającego - wyjątek). Prolog to deklaracja xml. 

        <?xml version="1.0" encoding="UTF-8"?>

    Prolog zawiera informacje o wersji oraz kodowaniu znaków w pliku xml.


    Element, atrybut i wartość


    Przeanalizujmy zawartość przykładowego pliku comics.xml:

        <?xml version="1.0" encoding="UTF-8"?>
    <comics>
        <universum type="marvel">
            <hero>
                <name>Spider-Man</name>
                <main-color>blue &amp; red</main-color>
                <gender>MAN</gender>
            </hero>
        </universum>
        <universum type="dc" />
    </comics>

    Prześledzimy interesujące nas fragmenty pliku… Oto one…

    Element - to podstawowa jednostka w dokumencie xml.

    Przykładowy element w dokumencie xml:

        <name>Spider-Man</name>

    Element składa się z:

    • znacznika otwierającego <name>,
    • znacznika zamykającego </name>,
    • zawartości, czyli Spider-Man.

    Element główny (ang. root element) jest to jedyny element w dokumencie xml:

        <comics>...</comics>

    Elementy dzieci (ang. child elements) są to elementy zagnieżdżone względem elementu rodzica (ang. parent element):

    W zaprezentowanym przykładzie, elementy dzieci to…

        <name>Spider-Man</name>
    <main-color>blue &amp; red</main-color>
    <gender>MAN</gender>

    dla, których rodzicem jest elemnent…

        <hero>...</hero>

    Atrybut - niesie dodatkowe informacje. Opisuje zawartość elementu.

    Przykładowe zastosowanie:

        <universum type="marvel">...</universum>

    Rozłożymy atrybut na czynniki pierwsze:

    • type to nazwa atrybutu,
    • "marvel" to wartość atrybutu (wartość umieszcamy w znakach " lub '),
    • = to znak równości.

    Pusty element

    Przykład praktycznego zastosowania pustego elementu:

        <universum type="dc"/>

    lub

        <universum type="dc"></universum>

    Komentarze


    Możesz stosować komentarze w plikach xml. Komentarz umieszczamy pomiędzy następującymi znakami: <!-- i -->. Oto przykład:

        <name>Spider-Man</name><!-- To jest mój ulubiony bohater! -->

    Predefiniowane encje - po co to jest?


    Niektóre znaki mogą sprawiać problemy: &, <, >, ", '. Stosuj zamienniki.

    • &amp; to znak &,
    • &lt; to znak <,
    • &gt; to znak >,
    • &quot; to znak ",
    • &apos; to znak '.

    Stosuj je we wnętrzu pliku xml w razie potrzeby!

    Wyświetlanie elementu w postaci tekstu


    Możesz zabronić procesorowi xml interpretowania konkretnego fragmentu. To jest troszkę dziwna składania, ale powinniśmy umieścić interesujący nas fragment pomiędzy <![CDATA[ a ]]>.

    CDATA - (unparsed) Character Data (nieparsowane dane znakowe).

        <?xml version="1.0" encoding="UTF-8"?>
    <tutorial>
        <lesson number="5">
        <title>Instrukcje warunkowe w języku C</title>
                    
        <content>
        <![CDATA[
        #include <stdio.h>
                    
        int main(void) {
            int digit = 12;
                    
            if(digit > 0 && digit % 2 == 0) {
            printf("Liczba %d jest dodatnia i parzysta! \n", digit);
            }
                    
                printf("Koniec programu...");
                return 0;
            }
        ]]>
        </content>
        </lesson>
    </tutorial>
